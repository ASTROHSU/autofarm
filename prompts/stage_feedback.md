# Feedback：差異分析與規則提取

## 你的任務

使用者已自行修改了 Draft 產出的摘要，現在把修改後的版本丟回來。
你要做的事：

1. **對照兩個版本**，逐段找出差異
2. **分析每個差異**，判斷它屬於哪一類修改
3. **提取可操作的規則**，寫成下次 Draft 可以直接套用的格式
4. **更新 draft-lessons.md**，把這次的對照記錄和規則存進去

---

## 差異分類

分析時，把每個改動歸類到以下其中一類：

| 類別 | 說明 | 例子 |
|------|------|------|
| 用字偏好 | 換了特定詞彙或句型 | 把「然而」換成「不過」 |
| 語氣調整 | 整體語感的改變 | 更口語、更直接、刪掉修飾語 |
| 結構調整 | 句子順序或段落切法改變 | 把第二句移到第一句 |
| 刪減 | 使用者覺得多餘的內容 | 刪掉括弧短評、刪掉某個說明 |
| 新增 | 使用者加入 Claude 沒寫的東西 | 加了某個數字、加了轉折 |
| 事實修正 | 使用者更正了不準確的內容 | 改了數字、改了組織名稱 |

---

## 輸出格式

```
## 這次的差異分析

### 主要改動

1. [改動描述]
   - Claude 版本：「...」
   - 使用者版本：「...」
   - 類別：[用字偏好 / 語氣調整 / 結構調整 / 刪減 / 新增 / 事實修正]
   - 推測原因：[為什麼使用者這樣改？]

2. [改動描述]
   ...

### 提取到的規則

（這些規則下次 Draft 會直接套用）

- [規則一，具體可操作]
- [規則二，具體可操作]
- [規則三，具體可操作]
```

輸出完分析後，**自動把這次的記錄寫入 draft-lessons.md**（包含：Claude 原版、使用者修改版、提取到的規則）。

最後說：「已更新 draft-lessons.md，下次 Draft 會套用以上規則。」

---

## 注意

- 不要評論使用者的修改「好不好」，只要客觀分析差異
- 規則要寫得具體，例如「避免用『然而』，改用『不過』」，不要寫「語氣更自然」
- 如果使用者改動很多，優先提取出現超過一次的模式，這才是真正的偏好
